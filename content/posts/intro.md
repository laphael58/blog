# 웹해킹과 보안 전문가로 가는 길 — 나의 시작과 목표

## 👋 시작 계기
나는 평소 개발에 관심이 있었고, 대학에 입학한 후 보안 분야에 큰 매력을 느끼기 시작했다.  
특히 웹해킹을 공부하면서 "취약점을 찾고 직접 분석하며 방어까지 구현하는 과정"이 너무 흥미로웠다.

또한 나의 장기 목표는 **Toss 보안 직군(Security Engineering / Web Security)**에 합류하는 것이다.  
이 목표를 이루기 위해서는 실전 중심의 학습과 강력한 포트폴리오가 필수적이라고 판단했다.

그래서 나는 지금부터 **웹해킹 + 웹개발 + 보안엔지니어링**을 함께 공부하며, 성장 과정을 이 블로그에 기록하려 한다.

---

## 🎯 최종 목표
내가 도달하고 싶은 구체적인 목표는 다음과 같다.

1. **Toss 보안 직군 취업**
2. **석사 진학 후 전문연(전문연구요원) 준비**
3. **내가 직접 만든 웹서비스에 보안을 적용하는 실전 프로젝트 수행**
4. **웹해킹 실습·취약점 공격/방어 분석 정리**
5. **블로그를 '실전 중심 보안 포트폴리오'로 완성**

---

## 🔥 학습 방식 & 블로그 운영 계획

### 1) 학습 방식
나는 하루하루 조금씩 하는 것보다  
**일주일 1~2일 “몰아치기 공부(8~12시간 집중)”**가 훨씬 잘 맞는다.  

따라서 블로그는 **매주 1개의 핵심 학습 글**을 올리는 방식으로 운영할 것이다.

---

### 2) 블로그 → 나중에는 직접 만든 사이트로 이전
나는 처음에는 Git 기반 블로그(Hugo/Jekyll)로 시작하지만  
나중에 웹 개발 실력이 쌓이면 **직접 만든 웹사이트**로 블로그를 이전할 계획이다.

그 사이트는 단순 블로그가 아니라:

- 로그인/회원가입 기능  
- 글 작성 기능  
- 파일 업로드  
- 관리자 페이지  

등을 갖춘 **실전 웹서비스 형태**로 만들고,  
일부러 취약점을 넣고, 내가 직접 방어하는 과정을 기록할 것이다.

즉, 이 블로그는 앞으로 **“내가 직접 만든 웹 보안 실습 플랫폼”**으로 진화할 예정이다.

---

## 🗓 1주차 학습 요약 (Reflected XSS / Stored XSS)

이번 주에는 웹해킹의 기초 중에서도 가장 중요한 공격인 **XSS**(Cross-Site Scripting)를 다뤘다.

### ✔ Reflected XSS 실습
- 사용자의 입력값이 서버에 반영되어 페이지에 출력되는 구조 분석  
- `<script>` 태그 기반의 기본 페이로드 테스트  
- 입력 필터링 미비를 이용한 코드 실행  
- URL 기반 XSS 재현

### ✔ Stored XSS 실습
- 게시판/댓글/프로필 등 DB에 저장되는 XSS 동작 확인  
- 스크립트가 영구적으로 저장되어 여러 사용자에게 전파되는 과정 분석  
- 피해 시나리오(세션 탈취, CSRF 연결, 백도어 스크립트 등)

### ✔ XSS 필터 우회 개념 기초
- HTML 엔티티 인코딩 우회  
- Attribute 기반 페이로드  
- Event Handler 기반 페이로드  
- DOM XSS 기본 구조 이해

→ 이 내용들은 다음 글에서 자세한 실습으로 다룰 예정이다.

## 🚀 앞으로의 계획
앞으로의 학습 계획은 다음과 같다.

- 2주차: **SQL Injection - 기초 + 실습 + 우회**
- 3주차: **CSRF / 인증 우회 / 세션 공격**
- 4주차: **파일 업로드 취약점 / RCE**
- 5주차: **내가 만드는 웹서비스 기초 구조 설계**
- 장기: **웹 서비스 개발 → 취약점 삽입 → 보안 적용 → 포트폴리오 완성**
